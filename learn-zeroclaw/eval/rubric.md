# Baseline Eval Rubric (v1)

## 1. 任务成功判定

- `pass`：满足任务目标，关键约束未违反
- `partial`：完成部分目标，但存在明显遗漏
- `fail`：未完成目标或输出不可用

## 2. 质量评分（1-5）

- `5`：正确、完整、结构清晰、无明显冗余
- `4`：基本正确，轻微遗漏或表达问题
- `3`：有可用内容，但关键点不完整
- `2`：偏离目标或错误较多
- `1`：无效输出

## 3. 失败类型枚举

- `timeout`：超时
- `tool_error`：工具执行失败
- `auth_error`：鉴权失败
- `network_error`：网络错误
- `validation_error`：输入/解析失败
- `unknown`：其他

## 4. 人工复核规则

- 每个任务至少抽样 1 次人工复核
- 成功率边界任务（pass/fail 不明显）必须复核
- 争议结果按“更保守”口径处理
